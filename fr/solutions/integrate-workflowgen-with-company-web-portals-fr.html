Intégration de WorkflowGen dans un portail Web d'entreprise

Utilisez les puissantes fonctionnalités d'intégration de WorkflowGen pour automatiser les processus tout en exploitant les infrastructures logicielles existantes de votre entreprise

Les fonctionnalités innovantes d'intégration de WorkflowGen peuvent être utilisées pour créer d'innombrables types d'intégrations. Un exemple pertinent de ce type d'intégration est le cas d'utilisation de WorkflowGen intégré dans les portails agent et client d'une compagnie d'assurances; cette intégration utilise les services Web, les URL WorkflowGen, l'application de workflow SENDMESSAGE et l'API GraphQL.

La vidéo ci-dessous présente comment ce type d'intégration est créée, comment elle fonctionne ainsi que l'expérience utilisateur des agents et des clients :



Dans un premier temps, le processus est créé dans WorkflowGen afin d'optimiser et d'automatiser le workflow entier, du début à la fin, ceci assurant qu'aucune étape ne sera oubliée. De plus, l'application SENDMESSAGE peut envoyer au client une notification SMS lorsque sa demande de réclamation sera approuvée ou refusée. Elle peut également être utilisée pour d'autres types de notifications telles qu'une demande d'informations supplémentaires sur une réclamation, ce qui permet une communication immédiate et en temps réel entre la compagnie et le client.



Le processus de réclamation est configuré dans le designer visuel de workflow de WorkflowGen



Le client se connecte à son portail, saisit les informations sur la réclamation et soumet la demande. Un service Web de WorkflowGen est ainsi enclenché et achemine la demande de réclamation à travers le workflow du processus tout en gérant les données de processus y compris les pièces jointes, telles que des reçus.



Le portail client affiche le statut de la demande de réclamation



De la même façon, l'agent se connecte à son portail comme d'habitude. Quand il clique sur la demande réclamation, le portail utilise une URL WorkflowGen pour ouvrir une fenêtre contextuelle dans laquelle est affiché le formulaire WorkflowGen.



Le portail agent avec la fenêtre contextuelle de WorkflowGen affichant le formulaire de la demande



L'agent consulte les détails de la réclamation dans le formulaire WorkflowGen



L'agent effectue ses actions de processus directement depuis le formulaire WorkflowGen



Chaque fois que le client ou l'agent retourne sur son portail, l'API GraphQL récupérera les données les plus récentes de WorkflowGen, ce qui assure que toutes les informations affichées soient toujours à jour. Aussitôt que la demande est traitée, l'application de workflow SENDMESSAGE envoie une notification texte SMS au client afin de l'informer que sa demande a été approuvée ou refusée.

Ce type d'intégration est un autre exemple de la façon dont une implémentation de WorkflowGen peut améliorer l'expérience utilisateur du début à la fin, aussi bien pour les clients que pour les employés.
