创建工作流 使用高级流程设计功能

用无码图形设计器建立复杂的工作流程

一个流畅易懂的HTML 5工作流程设计器结合任务、转变、同步和子工作流管理，自动化实现人机结合进程。

快速设计网络表单，无需编程

WorkflowGen包含内置的网络表单设计器，让非开发人员轻松创建专业的网络表单。表单设计器以HTML和JavaScript技术为基础，无需插件或附加软件。如果需要更高级的功能和定制化服务，表单设计器还包含一个.NET 代码编辑器。

使用综合性工作流门户吸引流程参与者

在单个网页视图中，流程参与者可发布新请求并实时监控其工作流的状态。根据WorkflowGen的配置，单点登录可实现自动化和无缝认证。

使用企业流程管理功能配置工作流

流程设计器可以在安全文件夹内组织流程便于分散，并对每个文件夹委派流程管理。WorkflowGen支持工作流的版本管理，使流程设计器实时更新工作流。

数秒内审批几十个请求

快速大量审批功能可在数秒内处理几十个请求。用户可根据流程数据组织请求，并创建定制视图在用户门户界面展示以便直接访问。

轻松点击就可跨多个流程执行工作流操作

在快速视图模式下，用户只需轻松点击就可从一个表单切换至另一个表单，并精确地查看请求状态，几下点击就可执行操作，通过单一视图窗口进行高级直观的导航。

轻松管理流程所用的全局列表

轻松管理流程所用的全局列表。企业相关数据不一定在ERP或CRM系统中展现，这些数据可以将管理分权至这些数据本身的管理员，从而丰富公司的工作流。全局列表管理使得特许用户通过WorkflowGen网络管理模块轻松创建和管理数据列表，以便在多个流程表单中使用。

轻点鼠标视觉化设置控件

交互式工作流视图提供了创建表单流程的最直观的方式。它使得流程设计器根据每个工作流操作将表单行为可视化。交互式工作流视图与表单设计器无缝整合。

简便的报告生成

高级报告功能使终端用户能运用流程数据创建自定义报告和仪表盘。

通过email完成任务

流程参与者可通过邮件收到预先定义的活动信息，且工作流程操作可在包含附件的HTML邮件通知中直接完成。

以全面多语种的企业解决方案在国际上扩张

流程参与者可以在门户界面直接更改语言、时区和密码。多亏了双字节合规及可编辑的资源文件，才能轻松支持多语言表单管理。

使用手机软件接入任意移动设备

使用WorkflowGen 移动版，用户可以通过其iOS和安卓设备远程访问工作流门户，执行工作流操作。

参与者管理

通过将流程参与者定义为单一用户，群组，协调者或目录，管理员可轻松管理流程参与者。全局参与者和基于角色的工作流参与者可由多个工作流共享，还可使用多个配置文件授权使用该产品的各项功能。

管理员还可允许流程主管重新指派操作，取消请求，查看统计资料，对特殊请求限定流程主管的范围，以及授予用户评论的权利。




目录管理

管理员可以使用目录连接器（活动目录，CSV文本文件，LDAP本地查询，文本文件）将用户同步化。

WorkflowGen包含的企业功能可以便于庞大人口的管理（自动目录同步化，创建多个目录，群体管理，层级管理，查询现有目录以便动态创建虚拟群体，用于已删除用户的工作流历史管理，关于用户活动的详细报告，用户替换）。




工作流应用程序

WorkflowGen及其内置工作流应用程序可将流程系统操作自动化，如从XML文档向数据库输出数据，从目录中检索用户信息，通过Twilio平台发送SMS通知等等。

开发人员可创建webhooks，网络服务或.NET程序集，作为定制工作流程序使用，实现广泛的集成可能性，如在工作流中使用第三方程序向数据库或DMS输出流程和表单数据，动态生成来自工作流数据的文档，并对短讯和监控系统发出警告和通知。




Web API

使用WorkflowGen GraphQL API， 可轻松实现与现有系统的集成。开发人员甚至能使用现有开发框架创建自己的工作流入口或特定用户界面/用户体验。

大多数运行时间和设计时间的工作流操作可通过应用程序接口进行操作，如检索请求，操作数据和状态；发布新请求；完成工作流操作；管理用户、群体、参与者和代表团；以及输入/输出流程定义。




认证及安全性

完成支持使用活动目录的Windows身份认证进行透明身份验证。您还能配置适用的认证（认证信息在数据库内处理），或自定义认证（自定义.NET HTTP 模块支持，与现有的单点登录系统集成）。

WorkflowGen支持 HTTPS SSL 数据加密。本产品在开发过程中遵循OWASP推荐规范，完全整合了网络应用安全技术。




性能及可扩展性

WorkflowGen 提供高度可扩展的环境，旨在应对频繁使用的情况。优化数据库结构和大小，并使用文件或数据库系统保存文件数据。

管理员可以在负载平衡的网络场环境下部署WorkflowGen，并为web应用使用多个服务器。可以使用其他数据库实例（实时复制）处理只读的SQL查询，以便减少服务器响应时间（数据库缩放功能）。
