Benefits

Paper forms are eliminated, saving time and resources while ensuring the integrity of process data.


Users can set workflow deadlines and easily track the processing of requests.


Tight and seamless integration of WorkflowGen into the existing iManage Work interface lets users continue to work in a familiar environment while providing new functionalities in both the foreground and the background.


Documents can be routed to other approvers or sent back to the requester for modifications.


Documents can be stored in predefined folders as configured in the workflow applications.


Existing document metadata can be updated dynamically over the entire workflow.


Email notifications keep users up-to-date on their requests over the entire course of the workflow, and include the form archive, a link to open the document directly within iManage, and links to document metadata.


Client Interface

Users can launch new workflow processes directly from iManage's client interfaces (FileSite, DeskSite, WorkSite Web and SharePoint) using the standard "right-click" menu functionality against the selected documents.




Bidirectional Integration

Bidirectional integration allows visibility into the iManage document repository from within a workflow application, allowing users to search and attach documents directly from the repository.




Process Management

The WorkflowGen End User Module can be embedded with the WorkSite Web or SharePoint environment to provide users with easy access to launch new processes or manage ongoing processes and actions to-do.




URL Calls

Additionally processes can be called directly from a URL allowing highly customised portal environments to display specific WorkflowGen processes within link lists.
